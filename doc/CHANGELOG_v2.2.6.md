# v2.2.6 更新日志

发布日期：2026-04-02

## 🔧 优化

- 重构桌面编辑器内部结构：将主控制器拆分为文档加载、局部修复、导出三块服务，减少 `editor_controller` 单文件堆积的业务逻辑。
- 重构编辑器画布结构：将 `graphics_view` 拆分为图层更新、区域渲染、输入交互三块模块，压短调用链，后续维护和排查更直接。
- 引入统一的编辑器会话与文档快照模型，加载、清理、导出前持久化都改为走统一状态入口，减少多处重复持有状态。
- 收口图片转换链路，统一 `PIL / numpy / QImage` 的桥接方式，降低编辑期图片对象重复转换和重复常驻。
- 优化桌面编辑器局部修复逻辑，连续涂抹时只应用最新一次修复结果。
- 优化编辑器缓存策略：文本渲染缓存集中到协调器管理，并增加缓存上限、失效和清理逻辑。
- 优化大图显示内存：compare / mask / inpaint 预览改为走可降采样显示帧，降低预览层峰值占用。
- 优化底图数组复用：新增弱缓存复用基础 RGB 数组，减少局部修复时重复执行整图 `PIL -> numpy` 转换。

## 🐛 修复

- 修复命令行/无头离屏渲染时 Qt 字体目录未正确指向项目内置 `fonts/` 的问题；现在在 `offscreen` 模式下会自动补充 `QT_QPA_FONTDIR`，避免出现 `Cannot find font directory ... Qt6/lib/fonts`、字体发现失败及字体回退异常。
- 修复编辑器实时修复在快速操作时可能出现的结果回退问题。
- 减少编辑器蒙版修复过程中的额外卡顿。
- 修复编辑器区域拖动、旋转和白框调整后撤销不稳定的问题；几何类操作恢复为稳定快照撤销，移动后可正常 `Ctrl+Z`。
- 修复切图和清空编辑器后旧图片缓存未完全释放的问题，避免多张大图切换时内存持续抬高。
- 修复 compare 视图、mask 叠层和 inpaint 预览在降采样显示后可能出现的对齐风险，保持场景坐标一致。
- 修复导出链路里 base image / inpainted image 的重复快照与重复拷贝，减少导出期间额外内存占用。
- 修复导出时当前活动图片已切换导致旧文档结果误回写的问题，只允许仍然匹配当前源图时更新运行时缓存与导出快照。
- 修复蒙版绘制提交路径里找不到 controller 时的旧兼容回退逻辑，统一要求通过 `model.controller` 执行命令，避免无撤销直写。
- 修复字体管理器预览区在切换同家族不同样式字体时显示不准确的问题，避免预览错误合并为同一字体效果。
- 修复渲染器字体选择逻辑，按当前选中的具体字体文件取字形，正确区分不同字体家族及同家族下的不同样式。

## 📝 说明

- 本版本主要是桌面编辑器内核重构、内存管理收口和撤销稳定性修复，没有新增独立功能入口。
- 新增离屏编辑器回归验证，覆盖大图切换后的内存回收、compare 同步缩放、mask / inpaint 预览对齐和导出回写链路。
- 重点目标是降低编辑器内的状态分散、重复缓存和大文件切换时的内存峰值，同时把控制器与画布主文件从“超大单文件”拆回可维护结构。
