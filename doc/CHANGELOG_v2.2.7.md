# v2.2.7 更新日志

发布日期：2026-04-06

## ✨ 新功能

### 文本替换规则管理

- 新增可视化的文本替换规则管理系统，支持在翻译后、渲染前对译文执行自定义替换
- 替换规则配置文件：`examples/text_replacements.yaml`，支持三个分组：
  - `common`：通用替换，横排竖排都执行
  - `horizontal`：仅横排时执行
  - `vertical`：仅竖排时执行
- 每条规则支持：字面替换、正则替换、启用/禁用、备注
- 替换结果自动写入 JSON，编辑器导出时自动跳过（避免重复替换）
- 原硬编码的 CJK 竖排/横排字符映射（`CJK_H2V`/`CJK_V2H`）已外部化到该配置文件，支持用户自定义修改

### Qt UI：替换规则管理页面

- 侧边栏"数据管理"分组下新增"替换规则"导航入口
- 表格式编辑器，双击 ✓/✗ 切换启用和正则状态
- 支持 Ctrl 多选、Shift 连续选、全部选中
- 工具栏按钮根据选中行状态动态变化（"启用"↔"禁用"、"正则"↔"取消正则"）
- 支持切换到原始 YAML 编辑模式（带语法高亮），保存前校验语法
- 完整 i18n 支持（zh_CN、zh_TW、en_US、ja_JP、ko_KR、es_ES）

### 根据气泡排版（强制横排）

- 新增渲染设置开关 `bubble_layout_english`，开启后对所有语言启用气泡形状排版并强制横排
- 英文默认已使用此排版方式（根据气泡蒙版断行），此开关主要影响韩文、中文等其他语言
- 开启后程序会从原图提取气泡蒙版来决定断行位置，并强制横排渲染，无论原始文本方向如何
- 适用场景：将漫画翻译为韩文或其他语言时，希望文本跟随气泡形状横排排版

## 🔧 优化

- 收口英文渲染链路：移除独立 `manga2eng` / `manga2eng_pillow` 渲染入口，统一改为默认渲染器内部处理英文断句。
- 默认英文横排现在会先应用一套独立的英文断句方法，只负责补 `BR`，不再单独走另一套贴字与排版通道。
- 保留旧配置兼容：历史配置里如果仍然写 `renderer: manga2eng` 或 `renderer: manga2eng_pillow`，现在会自动映射到 `default`，避免旧预设直接失效。
- 同步清理桌面端渲染器选项与说明文案，避免 UI 中继续暴露已经移除的独立英文渲染器入口。
- 桌面编辑器新增文本框旋转角度编辑与吸附辅助能力，相关实现来自 PR `#138`；合入后又将角度输入入口统一调整到左侧“样式设置”中的常驻输入框，并移除右键菜单、旋转手柄右键和弹出输入框。

## 🐛 修复

- 修复竖排文本布局计算时 `QRawFont` 为每个 pixel_size 重复从文件加载字体导致内存持续增长的问题；改为同路径字体复用隐式共享拷贝 + `setPixelSize` 切换大小，单次布局计算内存开销从数百 MB 降至接近零。- 修复“自动识别竖排，但实际强制改成横排”时单区域仍被按旧方向压成单行的问题；现在这类区域允许重新换行。
- 统一单区域强制不换行判断，收口到 `auto_linebreak.py` 维护，避免入口放开后又被下游逻辑重新收紧。
- 调整无 `BR` 文本的初始断句估算方式：不再按 OCR 原始 `lines/texts` 数量起步，统一改为在 `auto_linebreak.py` 内按当前布局盒子的宽高比例估算初始段数。
- 统一 `strict`、`smart_scaling`、`balloon_fill` 三种排版模式的无 `BR` 断句逻辑：现在都会先按同一套“反推段数 + `solve_no_br_layout`”流程生成断句，再进入各自后续的缩字 / 边界 / 气泡约束处理，避免 `balloon_fill` 单独走不同断句分支。
- 修复英文默认断句接入统一渲染后字号被继续自动缩小的问题；命中这条英文断句路径时，现在会直接沿用 OCR / 区域字号，不再继续进入自动字体缩放。
- 修复英文断句回退路径在缺少原图气泡蒙版时过度切行的问题，兜底逻辑改为按文本框宽度合并单词，而不是退化成接近“一词一行”。
- 修复省略号和标点空格预处理过度收口的问题：渲染前不再删除作者手动输入的标点后空格，连续 ASCII `.` 只会按每 3 个归并为一个 `…`，不会再把多组省略号继续折叠少一组。
- 修复横排英文 `...` 被统一压成中文 `…` 并渲染成居中省略号的问题；现在横排会保留 ASCII 三点，竖排仍按省略号逻辑转换。
- 修复横排文本测量与实际渲染对连续空格处理不一致的问题；`calc_text_block_dimensions` / `calc_box_from_font` 现在会按原始行文本直接量宽，避免编辑器预览里出现“字号没变但文字看起来变小”的假象。
- 修复多个字体文件内部 family / style 名称相同导致字体管理预览和实际渲染可能选到同一字体的问题；现在预览会缓存字体注册结果，渲染器会优先按选中的字体文件路径取字形。
- 修复 Qt 文本渲染器在文字描边内侧和外侧产生黑色、灰色“脏边”和光晕的问题；修复了 Numpy `uint8` 溢出、路径并集问题，并统一了 Alpha 预乘（Premultiplied Alpha）插值渲染链路。
- 修复竖排文本模式下括号产生过大缝隙的问题；现已精准提取字符墨迹边界，并将括号等边界符号的占位高度和游标步进严格压缩为半宽（0.5em），同时保持逗号、句号等常规标点占用全宽并居中对齐，符合中文排版直觉。
- 修复后端渲染导出时因图片流被过早关闭而抛出 `ValueError: Operation on closed image` 导致进程中断的问题；现采用严格的生命周期判定及预加载策略，防止引用冲突导致渲染失败。
- 修复编辑器在使用画笔或铅笔在已有蒙版区域涂抹时无法再次触发局部修复（重绘）的问题；重构了增量修复与笔画强制修复的内部处理链路，消除了任务冲突与冗余代码。
## 📝 说明

- 本版本重点是后端文本渲染整合，没有新增独立功能入口。
- 英文默认渲染现在的行为是：先断句补 `BR`，再直接按当前区域字号渲染；目标是保留原有英文排版体感，同时去掉旧 `manga2eng` 的分叉维护成本。
- 中文 / 日文等常规渲染模式下，`strict`、`smart_scaling`、`balloon_fill` 现在也共享同一套无 `BR` 断句入口；差异只保留在断句之后的几何缩放与约束阶段。
- 如果你本地还在用旧配置文件里的 `manga2eng` 渲染器名称，不需要手动迁移，也可以正常继续使用。
