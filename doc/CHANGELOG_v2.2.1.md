# v2.2.1 更新日志

发布日期：2026-03-21

## ✨ 新功能

- **"气泡内居中"设置重命名并重构**：原"垂直居中"更名为"气泡内居中"，更准确反映其用途。重构后该设置仅在文字区域确实位于检测到的气泡内时生效；气泡外的文字（旁白框、音效等）不受影响，保持顶部对齐。渲染结果同步写入 `white_frame_rect_local`，确保编辑器打开时白框位置与最终渲染一致。
- **桌面端提示词/字体管理增强**：Qt UI 的“提示词管理”页新增提示词复制、重命名；“字体管理”页新增字体导入、删除。现在可以在桌面端直接完成提示词文件与渲染字体的日常维护，无需手动打开目录操作。

## 🐛 修复

- 修复“仅翻译（JSON）”和 `load_text` 回写 JSON 时会丢失 `skip_font_scaling` 标志的问题。
- 修复“仅翻译（JSON）”在关闭“覆盖已存在文件”时跳过逻辑不一致的问题：没有对应 `_original.txt` 时现在会直接跳过。
- 修复 Qt UI 翻译完成后的“是否在编辑器中打开结果”提示框触发范围过宽的问题：现在只在“正常翻译”“导入翻译并渲染”“替换翻译”三种模式下显示。
- 修复 Qt UI “提示词管理 / 字体管理”页面多语言切换后部分标题、按钮、状态文案和预览文案没有完整走 i18n 的问题。

## 🔧 优化

- 统一 `skip_font_scaling` 的解析与保留逻辑。
- 统一 CLI 覆盖检测入口，避免普通模式和子进程模式行为不一致。
- 优化 OpenAI 兼容接口的 API Key 校验逻辑：非 OpenAI 官方地址（如 Ollama、本地/局域网部署、自定义域名反代）现在支持留空 API Key，不再在 Qt UI 启动前被强制拦截。
- 安装脚本（步骤1）国内克隆源由 gh-proxy 改为 Gitee 镜像（`https://gitee.com/hgmzhn/manga-translator-ui`），更稳定可靠。
- 移除 `main.py` 中硬编码的 Hugging Face 镜像站及 SSL 验证禁用，避免影响非国内用户的正常下载。
- 提示词/字体管理页刷新时会尽量保留当前选中项；如果删除当前正在使用的提示词或字体，会同步清理配置中的对应路径，避免残留失效配置。

## 📝 说明

- “仅翻译（JSON）”删除 `_original.txt` 的时机仍然是 JSON 保存成功之后。
- 导出原文 / 导出翻译流程仍会写入 `skip_font_scaling=false`。
- 如果使用 OpenAI 兼容接口且地址不是 `https://api.openai.com`，可仅填写 `OPENAI_API_BASE` 与模型名，`OPENAI_API_KEY` 允许留空；是否真正需要密钥由对应接口自身决定。
